# ADR-003 — 데이터: 풍력만, 약 1,000장 스토리 기반 서브셋

**Status**: Accepted (2026-07-10)

## Context

원본 데이터셋은 100,616쌍(풍력 8K RGB 80,906 + 태양광 열화상 640×512 19,710)이다.
MVP 단계에서 전량 인덱싱은 다운로드·임베딩(CPU) 시간과 비용이 과다하고,
풍력과 태양광은 이미지 분포가 완전히 달라 임베딩·전처리 파이프라인을 이원화해야 한다.

## Decision

**풍력만, 약 1,000장을 "유즈케이스가 전부 시연되도록 역산한" 구성으로 선택 다운로드한다:**

| 구성 | 수량 | 존재 이유 |
|---|---|---|
| 성산 단지 특정 호기들의 연도별 기록 | ~300 | 이력 조회(UC-2)·시계열 비교(UC-5)는 같은 설비의 다년 기록이 있어야 성립 |
| 주요 결함 클래스 균형 샘플 | 클래스당 ~80 | 유사 검색(UC-1)의 사례 풀 |
| 와류발생기 결함 전량 | 105 | 희귀 결함 에스컬레이션(UC-4). 원본에도 105장뿐 |
| 테스트셋 (위에서 분리, 인덱스 제외) | ~50 | VQA 채점 (SPEC-06) |

태양광은 제외하고 확장 1순위 백로그로 남긴다.

## Consequences

- (+) 다운로드·인덱싱이 감당 가능, UC 5종이 전부 실데이터로 시연 가능
- (−) 검색 풀이 얕아 유사도 절대값이 전량 인덱스보다 낮게 나올 수 있음 (임계값 해석 시 참작)
- (−) 클래스별 정답률 분해(원 지시서 E2)는 표본 부족으로 불가 — 2차
- 인덱싱 파이프라인(SPEC-02)은 서브셋 전용 로직 없이 폴더만 바꾸면 전량에도 동작하게 유지한다
