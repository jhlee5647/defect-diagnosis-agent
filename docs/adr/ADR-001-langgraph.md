# ADR-001 — 오케스트레이터를 LangGraph로 구현

**Status**: Accepted (2026-07-10)

## Context

오케스트레이터(SPEC-04)의 루프 — 도구 선택 → 호출 → 충분성 판단 → 반복 → 자기검증 — 를
구현하는 방식이 두 가지 있었다: ① OpenAI function calling으로 직접 while 루프 작성,
② LangGraph(상태 그래프 기반 에이전트 프레임워크). 직접 구현이 더 단순하고 의존성이 적지만,
사용 중인 다른 프로젝트와의 스택 일관성과 프레임워크 숙련 가치가 있었다.

## Decision

**LangGraph를 사용한다.** 상태(State)에 evidence·trace를 두고, 해석/도구실행/충분성평가/종결을
그래프 노드로 구성한다. 라우팅 로직이 그래프 구조에 드러나야 한다는 원 요구사항을 그래프 정의
자체로 충족한다.

## Consequences

- (+) 루프·분기·상태가 그래프로 구조화되어 trace 수집이 자연스럽다
- (+) 다른 프로젝트(LangGraph 기반)와 스택 일관성
- (−) 직접 구현 대비 학습 계층이 하나 더 — 디버깅 시 프레임워크 내부를 이해해야 할 수 있음
- (−) 의존성 증가. 프레임워크 마이그레이션 비용 발생 가능
- 기각한 대안: 직접 구현 — 더 단순하지만, 상태 관리·로그 구조화를 손수 만들어야 함
