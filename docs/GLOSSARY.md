# 용어집

스펙·ADR에서 쓰는 용어 정의. 스펙을 읽다 모르는 단어가 나오면 여기를 본다.

## 도메인 (풍력 점검)

| 용어 | 뜻 |
|---|---|
| 단지 / 호기 | 풍력발전소 이름(성산, 강원 등) / 그 안의 발전기 번호 (예: "성산 5호기") |
| 블레이드 | 발전기 날개. 한 발전기에 A·B·C 3장 |
| 부위 (side) | 날개의 면: Leading Edge(앞전), Trailing Edge(뒷전), Pressure Side(압력면), Suction Side(흡입면) |
| 결함 카테고리 9종 | 오염, 테이프 손상, 페인트 손상, 라미네이트 노출, 라미네이트 손상, 접합부 크랙, 리셉터 손상, 레인카라 결함, 와류발생기 결함. 페인트 손상이 가장 흔하고(약 40%) 와류발생기가 가장 희귀(0.1%) |
| severity (심각도) | 결함의 위험도 1~4. 1=경미한 관찰 대상, 4=즉시 조치. 정상 사진에는 없음 |
| O&M | 발전소 운영·정비(Operation & Maintenance) 업무 |

## 데이터 (사진 + 라벨 JSON)

| 용어 | 뜻 |
|---|---|
| 라벨 JSON | 사진 1장마다 딸린 정답 파일. 설비 정보 + 결함 목록 + 설명문 + 4지선다 문제가 들어 있음 |
| annotation | JSON 안의 "결함 1개 기록". 위치(bbox)·종류(category_id)·심각도(severity)를 가짐. 결함이 3개면 annotation도 3개, **정상 사진이면 아예 없음** |
| bbox | 결함 위치를 표시하는 직사각형. `[x, y, 가로, 세로]` 네 숫자 (COCO 방식). x,y는 왼쪽 위 모서리 |
| COCO | 이미지 라벨링의 사실상 표준 포맷 이름. "bbox가 COCO 방식"이면 [x,y,w,h] 순서라는 뜻 |
| 원본 좌표계 | 사진 원본(8256×5504 픽셀) 기준의 좌표. annotation의 bbox가 이 기준 |
| cropped_bbox | 원본에서 잘라낸 1920×1080 확대 영역의 위치. 4지선다 문제의 좌표 보기는 **이 영역 기준**이라 원본 좌표와 변환이 필요 |
| VQA | Visual Question Answering. 데이터에 내장된 4지선다 문제(결함 유무/위치/특징/유형). 우리 시스템의 정답률을 채점하는 "내장 모의고사" |
| object_description | 전문가가 쓴 사진 설명문(한국어 여러 문단). 검색 코퍼스와 few-shot 재료로 씀 |
