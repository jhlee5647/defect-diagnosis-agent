## 1. 스토리 — 이 모듈이 하는 일

**무엇을 만드나 (성격 규정 먼저):**
- **프론트엔드 개발이 아니다.** HTML/JS를 쓰지 않는다. Gradio는 파이썬 몇십 줄로
  "입력칸·버튼·출력칸"을 선언하면 웹페이지를 자동 생성해주는 라이브러리다. demo.py의 실체는
  SPEC-04의 `diagnose()` 함수를 웹 화면에 연결하는 **어댑터 수십 줄**이다.
  비유: 오케스트레이터가 엔진, demo는 계기판과 핸들 — 자체 로직이 없어야 한다.
- **테스트용이 아니라 실사용 화면이다.** 엔지니어가 사진을 올리고 질문하는 이 시스템의
  공식 입구이자 최종 산출물의 하나다. 자동 채점(테스트)은 SPEC-06 eval의 일이고 demo는
  채점하지 않는다. 단 MVP라 로컬 실행이며, 서버 배포·계정·동시 접속은 비스코프.

이 시스템의 가치는 최종 답이 아니라 **답에 도달하는 과정**에 있다 — "어떤 도구를 왜 골랐고,
뭘 관찰했고, 왜 부족하다고 판단해 다시 검색했는가." 데모는 그 과정을 사람이 눈으로
따라갈 수 있게 만드는 창구다. 화면 구성:

```
┌─────────────────────────────────────────────────┐
│  [사진 업로드(선택)]   [질문 입력창]        [실행]  │
│  [예시 버튼: 표준진단 | 이력조회 | 지식질문 | …]     │
├─────────────────────────────────────────────────┤
│  추론 과정                                        │
│   해석: 진단 의도, 필요 정보 4가지 …               │
│   반복1: vlm_analyze + visual_search 선택         │
│          이유: 아직 아무도 사진을 안 봤고 …          │
│          관찰: 오염 또는 라미네이트 노출, 확신 낮음   │
│          충분성: 부족 — 후보가 2개로 갈림            │
│   반복2: …                                       │
├─────────────────────────────────────────────────┤
│  최종 리포트 (6항목)  /  간결 답변                  │
└─────────────────────────────────────────────────┘
```

실행: `uv run python -m src.demo` → 브라우저에서 로컬 접속.

## 2. 함정 — 왜 규칙이 필요한가

**함정 1 — 결과만 보여주는 데모.** 최종 리포트만 표시하면 겉보기에 "고정 파이프라인 + 좋은 프롬프트"와
구분이 안 된다. 반복별 도구 선택 이유가 화면에 보여야 이 시스템의 차별점(질문마다 다른 경로)이 증명된다.

**함정 2 — 화면에 스며드는 판단.** 데모가 오케스트레이터의 반환을 "보기 좋게 고치는" 순간
(예: 확신 낮은 판정을 매끈하게 다듬기) 화면과 실제 시스템이 다른 것이 된다. 데모는 **표시만** 한다.

## 3. 실행 흐름

```
1. 입력  : 사진(선택) + 질문 텍스트. 예시 버튼 누르면 UC 5종의 프리셋 질문이 입력창에 채워짐
2. 호출  : diagnose(사진, 질문) — SPEC-04의 진입점 하나만 호출
3. 표시  : AgentResult.trace → 반복별 (도구 선택 이유 / 호출 파라미터 / 관찰 요약 / 충분성 판단)
           AgentResult.답변 → 리포트 또는 간결 답변
```

반복별 실시간 갱신(스트리밍)은 **구현 시 확정** — 완료 후 일괄 표시로 시작해도 충분.

## 4. 규칙 — 각각이 막는 사고

| # | 규칙 | 왜 (없으면 나는 사고) |
|---|---|---|
| R1 | 데모는 `diagnose()` 호출과 결과 표시만 한다. 판단·가공·재시도 로직 금지 | 함정 2 — 화면과 시스템의 불일치 |
| R2 | trace의 반복별 4요소(이유·파라미터·관찰·판단)를 전부 노출 | 함정 1 — 과정이 안 보이면 데모의 존재 이유가 없음 |
| R3 | 사진 없는 질문도 정상 동작 (이력 조회·지식 질문) | UC-2·3 시연 불가가 됨 |
| R4 | 에러·에스컬레이션도 그대로 표시 ("전문가 확인 필요" 포함) | 정직한 불확실성이 이 시스템의 정답 동작인데 화면이 숨기면 안 됨 |
