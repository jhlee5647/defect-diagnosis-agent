# 발전설비 결함 진단 — 멀티모달 Agentic RAG

풍력 블레이드 점검 사진을 진단하고 근거를 인용하는 단일 에이전트 RAG 시스템.
질문마다 도구(유사 사진 검색 · 설비 이력 조회 · 결함 기준 검색 · VLM 관찰)를 스스로 골라
조사하고, 근거가 부족하면 방법을 바꿔 재시도하며, 최종 답의 인용을 자기검증한다.

- 아키텍처·성공 기준: [docs/specs/SPEC-00-overview.md](docs/specs/SPEC-00-overview.md)
- 설계 결정 근거: [docs/adr/](docs/adr/)

## MVP 상태 (2026-07-14)

SPEC-00 성공 기준 3종 대비:

| 성공 기준 | 상태 |
|---|---|
| 1. E2E 데모 — 업로드 + 질의 → 추론 로그 → 6항목 리포트 (Gradio) | **달성** |
| 2. Agentic 증명 — UC 5종 경로 상이 + 재시도/보강 루프 1개 이상 | **부분 달성** — UC 4/5 PASS, 재관찰 루프 UC-1에서 동작. UC-4는 샘플 데이터 한계로 2차 과제 (아래) |
| 3. 미니 평가 — VLM 단독 vs few-shot 주입 정답률 비교표 | **달성** — 47% → 63% (+16%p) |

## 실행

```bash
uv sync                                        # 의존성 설치
uv run pytest                                  # 단위 테스트
uv run python -m src.ingest --data-dir data    # 인덱스 구축 (D1·V1·V2)
uv run python -m src.demo                      # 웹 데모 (Gradio)
uv run python eval/run_eval.py [--limit 5]     # 자동 평가
```

## 평가 결과 (2026-07-11, 샘플 데이터셋)

데이터: 풍력 블레이드 75장 (테스트셋 20장 층화 분리 — 벡터 인덱스에서 하드 제외, 인덱스 55장).
전체 결과: [eval/results.md](eval/results.md)

### VQA — 유사 사례 few-shot 주입 효과 (핵심 가설)

같은 크롭 사진·같은 4지선다 문항을 두 조건으로 채점 (gpt-4o, 버전 고정):

| 조건 | 결함유무 | 유형판별 | 위치 | 특징 | 전체 |
|---|---|---|---|---|---|
| (a) VLM 단독 | 60% (12/20) | 42% (8/19) | 47% (8/17) | 37% (7/19) | **47%** (35/75) |
| (b) 유사사례 주입 (RAG) | 85% (17/20) | 63% (12/19) | 41% (7/17) | 58% (11/19) | **63%** (47/75) |

**검색된 유사 사례를 주입하면 전체 정답률 +16%p (47%→63%)** — 결함유무 +25%p,
유형판별 +21%p, 특징 +21%p. 위치만 −6%p인데, 텍스트 few-shot(과거 사례의 라벨·설명문)은
좌표 판단에 재료가 되지 못하기 때문으로 보인다. 형식실패는 (a) 2건 vs (b) 11건 —
few-shot 블록이 길어지면 "한 글자만" 지시 위반이 늘어나는 트레이드오프가 관찰됨.
위치 문항 2건은 라벨 정답 좌표가 공유 변환 규칙과 불일치해 '라벨 오류'로 제외.

### 라우팅 검증 — UC 5종 통과 제약 (trace 기계 채점)

| UC | 판정 | 비고 |
|---|---|---|
| UC-1 표준 진단 | PASS | 4종 전부 + few-shot 재관찰 순서 충족 (프롬프트 개선 후) |
| UC-2 이력 조회 | PASS | history 단독 호출 (금지 도구 0) |
| UC-3 지식 질문 | PASS | knowledge만 사용 |
| UC-4 희귀 결함 | FAIL | 에스컬레이션 미발동 — 샘플 한계 (아래) |
| UC-5 시계열 비교 | PASS | history(과거 사진 경로)가 2장 비교보다 먼저 (프롬프트 개선 후) |

최초 실행은 2/5였다 — 검색 품질이 좋아지자 충분성 평가가 이력·기준 확보 전에 조기 종료.
충분성 프롬프트에 "리포트 6항목 재료 각각 확보 전 종료 금지"를, 선택 프롬프트에 비교 시나리오
힌트를 넣어 4/5로 개선. 남은 UC-4는 샘플 데이터 한계: 최희소 클래스(La Damage 1장)조차
흔한 클래스와 시각적으로 유사해 유사도·VLM 확신이 모두 높게 나오고, 그 상태에서 확신 있는
오진(La Damage→Paint Damage)이 발생한다. 확신 기반 에스컬레이션이 잡지 못하는 위험 유형이다.
해결 경로는 정의해 두었다: La Damage 사례를 인덱스에 여러 장 확보하면 유사 사진 검색 상위
결과에 클래스 경합이 생기므로, "1·2위 유사도 차가 작으면서 클래스가 서로 다름"을 에스컬레이션
발동 조건에 추가해 검증한다 ([eval/results.md](eval/results.md)). 현재 1장으로는 경합 자체가
재현되지 않아 2차 과제로 남긴다.
